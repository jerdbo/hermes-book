## 阅读效率优化技巧

### 1. 利用附录快速定位
- **附录 B（流程图汇总）**：需要图表时直接查找，无需翻阅全章
- **附录 C（术语表）**：遇到陌生术语时快速查询，避免中断阅读流
- **附录 D（本附录）**：根据角色选择最优路径，节省 50% 时间

### 2. 跳过与路径无关的细节
- **架构师路径**：可跳过 Ch-18-21 的 Rust 语言细节，直接看 Ch-22 的状态机设计
- **Rust 迁移路径**：可跳过 Ch-05-16 的 Python 实现细节，重点关注问题诊断（Ch-17）
- **记忆与学习路径**：可跳过 Ch-10（终端沙箱）、Ch-13-14（Gateway 平台）

### 3. 问题驱动阅读
全书识别了 65 个问题（P-01-01 到 P-34-XX），每个问题都有编号引用。如果你只关心特定问题：
- 先在 Ch-17 的问题分布矩阵中定位问题编号
- 查看"首次出现"列，跳转到对应章节
- 阅读该章节的"问题清单"部分（每章末尾）

### 4. 代码优先 vs 理论优先
- **代码优先**：直接跳到关键代码片段（每章的 `# 源码位置:行号` 注释），理解实现后再读设计论证
- **理论优先**：先读每章开头的"开篇问题"和"本章小结"，建立框架后再深入细节
